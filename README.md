# niri + DMS dotfiles

My [Niri](https://github.com/YaLTeR/niri) compositor and
[DankMaterialShell](https://github.com/AvengeMedia/DankMaterialShell) (DMS)
preferences, kept here so I can restore them when rebuilding any of my machines
(laptop, other laptop, desktop).

## Structure
```
niri/            -> ~/.config/niri/            (compositor config)
  config.kdl
  dms/*.kdl      keybinds, colors, layout, cursor, window rules, wallpaper blur
dankshell/       -> ~/.config/DankMaterialShell/
  settings.json  all DMS preferences (bar, dock, theme, lock/idle, etc.)
restore.sh       deploy everything onto a machine
```

## Restore on a rebuilt machine
```sh
git clone https://github.com/jhnhnsn/niri-config.git ~/niri-config
cd ~/niri-config && ./restore.sh
```
`restore.sh` copies the configs into place (backing up anything it overwrites to
`*.bak.<timestamp>`), validates the niri config, and enables the `dms.service`
systemd user unit (the canonical way DMS autostarts).

## Updating the backup (after changing settings on a machine)
```sh
cp ~/.config/niri/config.kdl        ~/niri-config/niri/config.kdl
cp ~/.config/niri/dms/{alttab,binds,colors,cursor,layout,windowrules,wpblur}.kdl \
   ~/niri-config/niri/dms/
cp ~/.config/DankMaterialShell/settings.json ~/niri-config/dankshell/settings.json
cd ~/niri-config && git add -A && git commit -m "Update preferences" && git push
```

## Per-machine notes
- **`dms/outputs.kdl` is NOT tracked** — monitor/display layout differs per machine.
  `restore.sh` leaves an existing one alone, or creates an empty file (niri then
  auto-detects). Configure with `niri msg outputs` after restoring.
- Some `dms/*.kdl` files (`layout.kdl`, `alttab.kdl`, `colors.kdl`) are
  auto-generated by DMS and/or derived from the current wallpaper, so they may be
  regenerated locally after restore — that's expected.
- A few keys in `dankshell/settings.json` are machine-specific (display profiles,
  GPU ids); they're empty/neutral here and DMS re-populates them per machine.
